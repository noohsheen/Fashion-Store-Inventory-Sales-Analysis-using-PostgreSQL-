# Fashion Store Inventory & Sales Analysis Using PostgreSQL

## Project Overview

The Fashion Store Inventory & Sales Analysis project focuses on analyzing retail fashion data using PostgreSQL. The dataset contains 50 fashion products with information such as category, brand, color, size, price, stock quantity, units sold, and customer ratings. The database is designed to store and manage this data efficiently while enabling meaningful business analysis.
The project aims to generate actionable insights that can help fashion retailers improve inventory management, identify top-performing products and brands, evaluate sales performance, and better understand customer preferences and sales trends.

## Objectives
 
* Design and manage a fashion retail database using PostgreSQL.
* Analyze product sales and inventory data.
* Identify top-selling products and best-performing brands.
* Calculate revenue generated by products and categories.
* Monitor stock levels and detect low-inventory products.
* Evaluate customer preferences using product ratings.
* Generate business insights to support decision-making.

## How to Use

### Step 1: Create the Database

Run the following command in PostgreSQL:

```sql
CREATE DATABASE fashion_store;
```

Connect to the newly created database.

### Step 2: Create the Table

Execute the table creation query to create the `products` table.

### Step 3: Insert Data

Run the INSERT statements provided in the SQL script to populate the table with fashion store product data.

### Step 4: Execute Analysis Queries

Run the SQL queries included in the project to perform:

* Data Exploration
* Sales Analysis
* Revenue Analysis
* Inventory Analysis
* Brand Performance Analysis
* Customer Preference Analysis

### Step 5: Review Results

Analyze the query outputs to identify:

* Top-selling products
* Revenue-generating categories
* Best-performing brands
* Highest-rated products
* Low-stock inventory items
* Inventory value by product
